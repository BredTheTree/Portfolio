Arduino DC Motor Controller

IR Remote Controlled DC Motor with Arduino

C++ Arduino Electronics IR Remote

📋 Project Overview

Summary

This Arduino project is a DC motor wired to an IR sensor, allowing its speed and direction to be controlled using a remote control

Key Features:

  • A DC motor capable of switching speed and direction at the press of a button
  • An IR sensor to pickup inputs from the remote
  • An LCD screen that displays the current speed and direction of the motor
  • A 9V battery to provide enough power to the circuit

Technical Specifications:

Microcontroller:
Arduino Uno R3
Motor Driver:
L293D
Motor Type:
9v DC motor with fan attatchment
Power Supply:
9v battery w/ power supply module + laptop
IR Receiver:
VS1838

💻 Source Code

arduino_motor_controller.ino
// Force IRremote to use Timer1 (avoids LCD timing conflict) #define IR_USE_TIMER1 #include <IRremote.hpp> #include <LiquidCrystal.h> // LCD pin setup LiquidCrystal lcd(12, 13, 7, 8, 9, 10); // Motor pins #define RECV_PIN 11 #define SPEED_PIN 5 #define DIR1 4 #define DIR2 3 // Motor state int mSpeed = 0; // Current motor speed (0–255) bool forward = true; // Direction: true = forward int lastSpeed = -1; bool lastDirection = false; void setup() { // Initialize LCD lcd.begin(16, 2); lcd.print("IR Motor Ready"); // Serial for debug Serial.begin(9600); Serial.println("IR Motor Control Active"); // IR setup IrReceiver.begin(RECV_PIN, ENABLE_LED_FEEDBACK); // Motor pins pinMode(SPEED_PIN, OUTPUT); pinMode(DIR1, OUTPUT); pinMode(DIR2, OUTPUT); } void loop() { // Check IR input if (IrReceiver.decode()) { unsigned long code = IrReceiver.decodedIRData.decodedRawData; if (code != 0) { // Ignore spurious zeros Serial.print("Code received: "); Serial.println(code, HEX); switch (code) { case 0xBA45FF00: // Power - stop motor mSpeed = 0; Serial.println("Motor stopped"); break; case 0xB946FF00: // Vol+ - increase speed mSpeed += 5; if (mSpeed > 255) mSpeed = 255; if (mSpeed < 155 && mSpeed != 0) mSpeed = 155; // prevent stall Serial.print("Speed increased to "); Serial.println(mSpeed); break; case 0xEA15FF00: // Vol- - decrease speed mSpeed -= 5; if (mSpeed < 155 && mSpeed != 0) mSpeed = 155; // prevent stall Serial.print("Speed decreased to "); Serial.println(mSpeed); break; case 0xBF40FF00: // Direction toggle forward = !forward; Serial.print("Direction: "); Serial.println(forward ? "Forward" : "Reverse"); break; default: Serial.println("Unknown button"); break; } } IrReceiver.resume(); // Prepare for next code } // Apply motor direction digitalWrite(DIR1, forward ? HIGH : LOW); digitalWrite(DIR2, forward ? LOW : HIGH); analogWrite(SPEED_PIN, mSpeed); // Update LCD only when speed/direction changes if (mSpeed != lastSpeed || forward != lastDirection) { noInterrupts(); // prevent LCD corruption lcd.setCursor(0, 0); lcd.print("Speed: "); // pad with spaces lcd.setCursor(7, 0); lcd.print(mSpeed); lcd.setCursor(0, 1); lcd.print("Dir: "); // pad with spaces lcd.setCursor(5, 1); lcd.print(forward ? "Forward" : "Reverse"); interrupts(); lastSpeed = mSpeed; lastDirection = forward; } }

Code Explanation:

The sketch is organized around input handling, motor output, and display updates. It listens for decoded IR commands, translates them into speed or direction changes, applies those values to the motor driver, and refreshes the LCD only when the state actually changes.

  • Setup initializes the LCD, serial monitor, IR receiver, and output pins so every hardware component is ready before the main loop begins.
  • Main loop continuously checks for remote input, updates the motor state, and writes the latest speed and direction to the display.
  • IR signal processing decodes raw button values from the remote and maps each recognized code to an action such as stop, increase speed, decrease speed, or reverse direction.
  • Motor control logic uses digital outputs for direction and PWM on the speed pin so the motor can reverse cleanly while still supporting variable speed control.

🔧 Implementation Details

One of the main implementation challenges was keeping multiple hardware components working together without conflicts. The project combines IR input, PWM motor control, and LCD output in a single loop, so the code was structured to avoid unnecessary display writes and to keep the interface responsive while the motor is running.

Another practical issue was making the motor behave reliably at lower speeds. The sketch enforces a minimum non-zero PWM value to prevent the fan motor from stalling, which makes the controls feel more consistent when adjusting speed from the remote.

Future Improvements:

  • Button labeling could be expanded so more remote keys trigger preset speed levels or additional operating modes.
  • Closed-loop feedback could be added with a sensor to measure actual RPM instead of estimating output from PWM alone.
  • Power management could be improved with a more robust external supply for longer demos and more stable motor performance.
